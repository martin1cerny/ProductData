Terms, definitions, and abbreviated terms

For the purpose of this document, the following definitions apply.

3.1 Terms and definitions

3.1.1 asset

item, thing or entity that has potential or actual value to an organization

[ISO 55000:2014, 3.2.1]

3.1.2 attribute

data element for the computer-sensible description of a property, a relation or a class

[ISO 22274:2013, 3.2]

3.1.3 configurable product

a product offered by a manufacturer, that can be configured and ordered based on the offerings of the manufacturer

3.1.4 configured product

a product offered by a manufacturer, that cannot be further configured. It is offered in exact one variant.

3.1.5 data sheet

populated data template

3.1.6 data template

schema providing a data structure used to describe the properties of objects

3.1.7 information

meaningful data

[EN ISO 9000:2015, 3.8.2]

3.1.8 Information structure

Machine readable schema for storing and transferring data

3.1.9 manufacturer

person or organization making offsite materials, products, components, and other items

[ISO 6707-1:2017, 3.6.3]

3.1.10 object

any part of the perceivable or conceivable world

[EN ISO 12006-2:2020, 3.1.1]

3.1.11 product

tangible outcome of a process

[ISO 6707-3, 3.3.1]

3.1.12 property

inherent or acquired feature of an object [ISO 6707-1:2017, 9.1.3]

3.1.13 requirement request

a collection of information that describes the construction objects, that are needed to fulfill a specific purpose in the design of a building

3.2 Abbreviated terms

CODV Construction Objects Data View specific model view definition used that store the IFC entities required to exchange information through data templates and data sheets information.
IFC specifies a conceptual data schema and an exchange file format for Building Information Model (BIM) data
PDS Product Data Sheets populated product data template
PDT Product Data Template data template relating to products
